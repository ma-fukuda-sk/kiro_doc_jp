# Hook Types（フックタイプ）

Kiro IDEのAgent Hooksで使用可能な各種トリガータイプと、それぞれのユースケースを説明します。

## トリガータイプ

### On Prompt Submit（プロンプト送信時）

ユーザーがプロンプトを送信したときにアクティブになります。

- シェルコマンド使用時、ユーザー入力は`USER_PROMPT`環境変数からアクセス可能
- **ユースケース**:
  - コンテキスト情報の追加
  - 問題のあるプロンプトのフィルタリング
  - プロンプトログの集中管理

### On Agent Stop（エージェント停止時）

エージェントがターンを完了してユーザーに応答した後に発火します。

- **ユースケース**:
  - エラーレポート付きコードコンパイル
  - コードフォーマット
  - エージェントによる変更のレビュー

### On File Create（ファイル作成時）

指定されたパターンに一致する新しいファイルがワークスペースに作成されたときにトリガーされます。

- **ユースケース**:
  - コンポーネントボイラープレートの生成
  - ライセンスヘッダーの追加
  - テストファイル構造の確立

### On File Save（ファイル保存時）

一致するファイルが保存されたときにアクティブになります。

- **ユースケース**:
  - リンターの実行
  - 依存ファイルの更新
  - ドキュメントの作成
  - テストの実行

### On File Delete（ファイル削除時）

一致するファイルが削除されたときにトリガーされます。

- **ユースケース**:
  - 関連ファイルのクリーンアップ
  - import文の更新
  - プロジェクト構造の維持

### Manual Trigger（手動トリガー）

オンデマンドでのhook実行を可能にします。

- **ユースケース**:
  - コードレビュー
  - ドキュメント生成
  - セキュリティスキャン
  - パフォーマンス最適化
