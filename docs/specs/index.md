# Specs

Specsは、開発プロセスを形式化する構造化されたアーティファクトです。高レベルのアイデアを明確な追跡と責任を持つ詳細な実装計画に変換するのに役立ちます。

## Specsとは？

Specsは以下を支援します：

- 要件を受け入れ基準を持つユーザーストーリーに分解
- シーケンス図やアーキテクチャ計画を含む設計ドキュメントの構築
- 個別タスクにわたる実装進捗の追跡
- プロダクトチームとエンジニアリングチーム間のコラボレーション

## クイックスタート

Specsは3フェーズのワークフローに従います：

1. **Requirements（要件）** → ユーザーストーリーと受け入れ基準を定義
2. **Design（設計）** → 技術アーキテクチャをドキュメント化
3. **Implementation（実装）** → タスクに分解して追跡
