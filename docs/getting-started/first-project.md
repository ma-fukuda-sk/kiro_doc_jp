# 最初のプロジェクト

このガイドでは、ハンズオンプロジェクトを通じてKiroのコア機能（steering files、specs、hooks、MCP servers）を包括的に説明します。

## 前提条件

- Kiroがインストールされていること
- 既存または新規のプロジェクトがあること
- プロジェクトの構造と技術スタックについて基本的な知識があること

## プロジェクトを開く

3つの方法でプロジェクトを開けます：

1. **File > Open Folder** を使用
2. フォルダをドラッグ＆ドロップ
3. コマンドラインから `kiro .` を実行

アクティビティバーのGhostアイコンをクリックしてKiroパネルにアクセスし、チャットセッションを開始します。

## Steering Files

Steering filesはプロジェクトコンテキストを提供し、Kiroがコードベース、規約、要件を理解するのに役立ちます。

### 機能

- steeringドキュメントを自動生成
- コーディング標準やチームのベストプラクティスを含むカスタムファイルを作成

詳細は[Steeringドキュメント](../steering/index.md)を参照してください。

## Specsシステム

Specsは高レベルのアイデアを実装計画に変換する機能で、3つのフェーズで構成されます：

1. **Requirements（要件）**: EARS記法による受け入れ基準を持つユーザーストーリー
2. **Design（設計）**: 技術アーキテクチャドキュメント
3. **Tasks（タスク）**: 個別の追跡可能な実装ステップ

詳細は[Specsドキュメント](../specs/index.md)を参照してください。

## Agent Hooks

ファイルの作成、保存、削除、または手動トリガー時に事前定義されたアクションを実行する自動化機能です。

### 設定方法

1. イベントタイプの選択
2. ファイルパターンの指定
3. 実行する指示の定義

詳細は[Hooksドキュメント](../hooks/index.md)を参照してください。

## Model Context Protocol (MCP)

MCPはKiroが以下にアクセスすることを可能にします：

- 専門知識ベース
- 外部API
- ドメイン固有のツール
- データベースやクラウドサービス

詳細は[MCPドキュメント](../mcp/index.md)を参照してください。
