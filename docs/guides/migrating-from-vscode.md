# VS Codeからの移行

KiroはVisual Studio Codeのオープンソース基盤上に構築されており、AI強化機能を追加しながらVS Codeからのスムーズな移行が可能です。このドキュメントでは、プロファイル移行、設定管理、拡張機能互換性について説明します。

## プロファイル移行

以下を含む開発環境を移行できます：

- 拡張機能
- テーマ
- 設定
- ショートカット

### 移行プロセス

1. **VS Codeからエクスポート**: コマンドパレット（`Cmd/Ctrl + Shift + P`）からプロファイルをエクスポート
2. **Kiroにインポート**: 同じインターフェースを使用してインポート

## 設定アーキテクチャ

KiroはVS Codeの設定を専用のAIコントロールで拡張しています。

### 設定へのアクセス

- **標準設定**: 「Preferences: Open Settings (UI)」
- **Kiro Agent セクション**: AI動作と自動化機能の管理用

## バージョン更新

「Kiroは定期的なリベースを通じてVS Codeの開発サイクルと同期を保ち」、AI機能強化とともに信頼性を維持するために安定したリリースを戦略的に選択しています。

## 拡張機能互換性

### レジストリ

KiroはVS Code Marketplaceではなく**OpenVSX拡張機能レジストリ**を使用します。

### 互換性のある拡張機能

OpenVSXで利用可能な以下の拡張機能はシームレスに移行し、AI統合により機能が強化されることが多いです：

- 言語拡張機能
- テーマ
- デバッグ
- Git拡張機能

### 重要な制限

**VS Code Marketplace限定の拡張機能は利用できない場合があります。**

「OpenVSXレジストリで利用可能な拡張機能のみインポートできます。」
