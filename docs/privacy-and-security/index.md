# プライバシーとセキュリティ

KiroはAWSベースのエージェント型IDEで、AWSインフラストラクチャ上に構築されたセキュリティを備えています。プラットフォームは共有責任モデルに従い、「クラウドのセキュリティ」はAWSの責任、「クラウド内のセキュリティ」はユーザーが管理します。

## URLフェッチ

KiroチャットモジュールにURLを貼り付けてコンテキストを取得できます。取得したコンテンツが適用される規約と法律に準拠していることを確認する責任はユーザーにあります。

## 操作モード

### Autopilot Mode（オートパイロットモード）- デフォルト

- ステップごとの承認なしで複数のステップを自律的に実行
- タスクの理解に基づいて決定を下す
- いつでもトグルまたは中断可能

### Supervised Mode（監視モード）

- ファイル操作前にユーザーの確認が必要
- 必要に応じて明確化の質問をする
- コード変更に対するユーザーコントロールを維持

ChatモジュールからファイルDの変更を表示、承認、元に戻すことができます。

## 信頼済みコマンド

デフォルトでは、すべてのコマンドは承認が必要です。以下を使用して信頼パターンを設定できます：

- **完全一致**: 例 `npm install`
- **ワイルドカード一致**: 例 `npm *`
- **全信頼**: `*`（注意して使用）

システムはコマンド文字列に対してシンプルなプレフィックスマッチングを使用します。

## セキュリティベストプラクティス

### リソースの保護

Kiroエージェントは以下にアクセスする可能性があります：

- ローカルファイル
- 環境変数
- AWS認証情報
- 設定ファイル

### 推奨事項

1. **ワークスペースの分離**と`.gitignore`の使用
2. **専用ユーザーアカウント**またはコンテナの使用
3. **最小権限の一時的なAWS認証情報**
4. **リポジトリ固有のアクセストークン**

## リモート拡張機能

KiroはSSHリモートを含むOpen VSX拡張機能をサポートしています。ただし、サードパーティの拡張機能はKiroによって管理されておらず、侵害されたリモートはローカルコードを実行する可能性があります。
